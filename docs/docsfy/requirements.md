# Markdown 静态站点服务需求（Golang）

## 1. 项目目标

使用 Golang 开发一个服务，将 Markdown 文档渲染为静态页面，并可通过 HTTP 直接访问，整体体验类似 docsify。

## 2. 核心功能需求

### 2.1 文档输入与目录结构

- 支持指定文档目录（默认 `docs/`）。
- 递归扫描目录下所有 `.md` 文件。
- 支持 `README.md` 或 `index.md` 作为首页内容。

### 2.2 Markdown 渲染

- 将 Markdown 渲染为 HTML 页面内容。
- 支持常用 Markdown 语法：标题、列表、链接、图片、表格、代码块。
- 代码块输出需包含可用于样式或高亮的 class。
- 标题应支持锚点，方便页面内跳转。

### 2.3 路由与访问

- 基于文档路径生成访问路由，例如：
  - `docs/guide/getting-started.md` -> `/guide/getting-started/`
- 首页映射到 `/`。
- 启动本地 HTTP 服务后可直接访问文档页面。
- 对不存在路由返回 404 页面。

### 2.4 页面结构与样式

- 提供默认页面布局：
  - 顶部站点标题
  - 左侧导航栏
  - 右侧内容区
- 导航根据目录结构自动生成。
- 提供基础响应式样式，支持桌面端和移动端访问。

### 2.5 静态资源

- 支持静态资源目录（如 CSS、JS、图片）托管与访问。
- 页面可加载默认主题样式文件。

### 2.6 构建与运行

- 提供命令行能力：
  - `serve`：启动本地服务进行浏览
  - `build`：将文档导出为纯静态站点文件
- `build` 输出目录默认 `dist/`，每个页面按路由导出为 `index.html`。

### 2.7 文档全文搜索

- 构建时生成搜索索引文件（如 `dist/search-index.json`）。
- 索引需包含至少以下字段：标题、路由、正文纯文本摘要（可截断）。
- 前端页面提供搜索输入框，支持关键词检索并展示匹配结果列表。
- 检索结果可点击跳转到对应文档页面。

### 2.8 增量构建与文件监听热更新

- `serve` 模式下支持监听文档目录与模板/资源目录文件变更。
- 当 Markdown 文件变更时，仅重建受影响页面与相关索引（避免全量重建）。
- 当样式或模板变更时，触发页面重新渲染并通知浏览器刷新。
- 提供稳定的变更处理机制（创建、修改、删除文件均可正确处理）。

## 3. 非功能需求

- 使用 Go 标准库优先实现 HTTP 服务与模板渲染。
- 代码结构需模块化，便于扩展（解析、渲染、服务分层）。
- 保持易用性：默认配置可开箱即用，支持命令行参数覆盖。
- 构建结果可部署到任意静态托管平台（Nginx、GitHub Pages、CDN）。

## 4. 建议目录结构

```text
cmd/main.go
internal/config
internal/content
internal/render
internal/site
internal/server
web/templates
web/assets
docs
dist
```

## 5. 验收标准（MVP）

- 可从 `docs/` 扫描并渲染 Markdown。
- 可通过 `serve` 在浏览器访问页面。
- 导航可反映目录层级。
- 可通过 `build` 生成可直接部署的静态站点产物。
- 关键模块具备基本单元测试（路由映射、渲染结果、导航生成）。

## 6. 验收标准（增强能力）

- 执行 `build` 后会生成可被前端检索使用的 `search-index.json`。
- 前端可根据关键词返回匹配文档并跳转。
- 在 `serve` 模式修改单个 Markdown 文件时，能够触发增量重建且页面可快速更新。
- 在 `serve` 模式修改模板或静态资源时，页面可自动刷新或重新加载后生效。
